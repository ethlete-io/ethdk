# Structured Data Binding

Reactive JSON-LD structured data management for Angular applications with automatic cleanup and type-safe Schema.org support.
Alternative to using the `StructuredDataComponent`.

## Overview

Structured data (JSON-LD) helps search engines understand your content and can enable rich results in search. Features include:

- **Reactive Updates**: Automatically updates when signal values change
- **Type-Safe**: TypeScript support for common Schema.org types
- **Multiple Instances**: Support for multiple structured data blocks per page

## Basic Usage

### Simple Structured Data

```typescript
import { Component } from '@angular/core';
import { applyStructuredDataBinding } from '@ethlete/core';

@Component({
  selector: 'app-about',
  template: '<div>About Us</div>',
})
export class AboutComponent {
  constructor() {
    applyStructuredDataBinding({
      '@context': 'https://schema.org',
      '@type': 'Organization',
      name: 'Acme Corporation',
      url: 'https://www.example.com',
      logo: 'https://www.example.com/logo.png',
    });
  }
}
```

### Reactive Structured Data

```typescript
import { Component, signal, computed } from '@angular/core';
import { applyStructuredDataBinding } from '@ethlete/core';

@Component({
  selector: 'app-article',
  template: '<article>{{ article().title }}</article>',
})
export class ArticleComponent {
  article = signal({
    title: 'My Article',
    author: 'John Doe',
    publishedDate: '2024-01-15',
    image: 'https://example.com/image.jpg',
  });

  constructor() {
    applyStructuredDataBinding(
      computed(() => ({
        '@context': 'https://schema.org',
        '@type': 'Article',
        headline: this.article().title,
        author: {
          '@type': 'Person',
          name: this.article().author,
        },
        datePublished: this.article().publishedDate,
        image: this.article().image,
      })),
    );
  }
}
```

### BreadcrumbList

```typescript
import { Component, inject } from '@angular/core';
import { Router } from '@angular/router';
import { applyStructuredDataBinding } from '@ethlete/core';

@Component({
  selector: 'app-page',
  template: '<div>Page</div>',
})
export class PageComponent {
  private router = inject(Router);

  constructor() {
    const segments = this.router.url.split('/').filter((s) => s);
    const baseUrl = 'https://example.com';

    applyStructuredDataBinding({
      '@context': 'https://schema.org',
      '@type': 'BreadcrumbList',
      itemListElement: segments.map((segment, index) => ({
        '@type': 'ListItem',
        position: index + 1,
        name: segment.charAt(0).toUpperCase() + segment.slice(1),
        item: `${baseUrl}/${segments.slice(0, index + 1).join('/')}`,
      })),
    });
  }
}
```

## Configuration

### Placement

Configure whether structured data is placed in `<head>` or `<body>`:

```typescript
import { provideStructuredDataConfig } from '@ethlete/core';

bootstrapApplication(AppComponent, {
  providers: [
    provideStructuredDataConfig({
      placement: 'body', // or 'head'
    }),
  ],
});
```

**Default**: `'body'` (Google's recommendation)

## Advanced Usage

### Multiple Structured Data Blocks

```typescript
import { Component } from '@angular/core';
import { applyStructuredDataBinding } from '@ethlete/core';

@Component({
  selector: 'app-blog-post',
  template: '<article>Blog Post</article>',
})
export class BlogPostComponent {
  constructor() {
    // Article structured data
    applyStructuredDataBinding({
      '@context': 'https://schema.org',
      '@type': 'Article',
      headline: 'Blog Post Title',
      // ... article properties
    });

    // Author structured data
    applyStructuredDataBinding({
      '@context': 'https://schema.org',
      '@type': 'Person',
      name: 'Author Name',
      // ... person properties
    });

    // Breadcrumbs
    applyStructuredDataBinding({
      '@context': 'https://schema.org',
      '@type': 'BreadcrumbList',
      // ... breadcrumb properties
    });
  }
}
```

### Dynamic Structured Data

```typescript
import { Component, signal, computed } from '@angular/core';
import { applyStructuredDataBinding } from '@ethlete/core';

@Component({
  selector: 'app-product',
  template: '<div>{{ product().name }}</div>',
})
export class ProductComponent {
  product = signal({
    name: 'Product',
    price: 99.99,
    rating: 4.5,
    reviews: 100,
    inStock: true,
  });

  constructor() {
    applyStructuredDataBinding(
      computed(() => {
        const p = this.product();
        return {
          '@context': 'https://schema.org',
          '@type': 'Product',
          name: p.name,
          offers: {
            '@type': 'Offer',
            price: p.price.toFixed(2),
            priceCurrency: 'USD',
            availability: p.inStock ? 'https://schema.org/InStock' : 'https://schema.org/OutOfStock',
          },
          aggregateRating: {
            '@type': 'AggregateRating',
            ratingValue: p.rating.toString(),
            reviewCount: p.reviews.toString(),
          },
        };
      }),
    );
  }
}
```

### Graph Structure

For complex relationships, use the `@graph` format:

```typescript
applyStructuredDataBinding({
  '@context': 'https://schema.org',
  '@graph': [
    {
      '@type': 'Organization',
      '@id': 'https://example.com/#organization',
      name: 'Example Organization',
      url: 'https://example.com',
    },
    {
      '@type': 'WebSite',
      '@id': 'https://example.com/#website',
      url: 'https://example.com',
      publisher: {
        '@id': 'https://example.com/#organization',
      },
    },
    {
      '@type': 'WebPage',
      '@id': 'https://example.com/page#webpage',
      url: 'https://example.com/page',
      isPartOf: {
        '@id': 'https://example.com/#website',
      },
    },
  ],
});
```

## API Reference

### Core Function

#### `applyStructuredDataBinding(binding: MaybeSignal<JsonLD.WithContext<JsonLD.Thing> | JsonLD.Graph | null | undefined>)`

Applies a structured data binding to the page.

**Parameters:**

- `binding`: JSON-LD structured data (can be a signal or computed)

**Returns:** `void`

**Cleanup:** Automatically removes the script tag when the component is destroyed.

### Configuration

#### `provideStructuredDataConfig(config: StructuredDataConfig)`

Configures global structured data behavior.

**Config Options:**

```typescript
type StructuredDataConfig = {
  placement: 'head' | 'body';
};
```

## Best Practices

### ✅ DO

- Use the most specific schema type available
- Include all required properties for your chosen type
- Use absolute URLs for images and links
- Use ISO 8601 format for dates
- Include `@context` in every structured data block

### ❌ DON'T

- Add structured data for content not visible on the page
- Include user-generated content without moderation
- Use relative URLs (always use absolute URLs)
- Duplicate information unnecessarily
