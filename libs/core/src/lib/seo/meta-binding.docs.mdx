# Meta Tag Binding

Reactive meta tag management for Angular applications with automatic cleanup, multi-instance support, and priority-based resolution.

## Overview

The meta binding utilities provide a comprehensive signal-based approach to managing meta tags. Features include:

- **Reactive Updates**: Automatically updates when signal values change
- **Multi-Instance Support**: Some tags (like og:image) can have multiple instances
- **Priority System**: Child routes override parent meta tags

## Basic Usage

### Simple Meta Tag

```typescript
import { Component } from '@angular/core';
import { applyDescriptionBinding } from '@ethlete/core';

@Component({
  selector: 'app-about',
  template: '<div>About Us</div>',
})
export class AboutComponent {
  constructor() {
    applyDescriptionBinding('Learn more about our company and mission.');
  }
}
```

### Reactive Meta Tags

```typescript
import { Component, signal, computed } from '@angular/core';
import { applyDescriptionBinding, applyKeywordsBinding } from '@ethlete/core';

@Component({
  selector: 'app-product',
  template: '<div>{{ product().name }}</div>',
})
export class ProductComponent {
  product = signal({
    name: 'Widget',
    description: 'A great widget',
    category: 'Tools',
  });

  constructor() {
    // Reactive description
    applyDescriptionBinding(computed(() => this.product().description));

    // Reactive keywords
    applyKeywordsBinding(computed(() => [this.product().category, this.product().name, 'shop']));
  }
}
```

## All-in-One Social Media

The most convenient way to set up all social meta tags at once:

```typescript
import { applySocialMediaBindings } from '@ethlete/core';

@Component({
  selector: 'app-article',
  template: '<article>...</article>',
})
export class ArticleComponent {
  article = signal({
    title: 'My Article',
    description: 'Article description',
    image: 'https://example.com/image.jpg',
    url: 'https://example.com/article',
  });

  constructor() {
    applySocialMediaBindings({
      title: computed(() => this.article().title),
      description: computed(() => this.article().description),
      image: computed(() => this.article().image),
      url: computed(() => this.article().url),
      openGraph: {
        type: 'article',
        siteName: 'My Blog',
      },
      twitter: {
        card: 'summary_large_image',
        site: '@myblog',
      },
      article: {
        publishedTime: '2024-01-15T12:00:00Z',
        tags: ['technology', 'programming'],
      },
    });
  }
}
```

## Configuration

### Multi-Instance Tags

Configure which meta tags can have multiple instances:

```typescript
import { provideMetaConfig, DEFAULT_MULTI_INSTANCE_TAGS } from '@ethlete/core';

bootstrapApplication(AppComponent, {
  providers: [
    provideMetaConfig({
      multiInstanceTags: new Set([...DEFAULT_MULTI_INSTANCE_TAGS, 'custom:tag']),
      transformer: (content: string, locale: string) => {
        // Custom transformation logic
        return content;
      },
    }),
  ],
});
```

### Content Transformer

Transform meta tag content based on locale or other factors:

```typescript
provideMetaConfig({
  transformer: (content: string, locale: string) => {
    // Example: Add locale-specific prefixes
    if (locale === 'de') {
      return `🇩🇪 ${content}`;
    }
    return content;
  },
});
```

## Advanced Usage

### Custom Meta Tags

```typescript
import { applyMetaBinding } from '@ethlete/core';

// Custom name-based meta tag
applyMetaBinding({
  name: 'custom-tag',
  content: 'custom value',
});

// Custom property-based meta tag (Open Graph style)
applyMetaBinding({
  property: 'custom:property',
  content: 'custom value',
});

// Allow multiple instances
applyMetaBinding({
  property: 'custom:multiple',
  content: 'value 1',
  allowMultiple: true,
  key: 'custom:multiple:1',
});
```

## API Reference

### Core Functions

#### `applyMetaBinding(binding: MaybeSignal<MetaTagConfig | null | undefined>)`

Low-level function for applying any meta tag.

#### `applyDescriptionBinding(binding: MaybeSignal<string | null | undefined>)`

Sets the page description meta tag.

#### `applyKeywordsBinding(binding: MaybeSignal<string[] | null | undefined>)`

Sets the keywords meta tag.

#### `applyAuthorBinding(binding: MaybeSignal<string | null | undefined>)`

Sets the author meta tag.

#### `applyRobotsBinding(binding: MaybeSignal<RobotsConfig | null | undefined>)`

Configures search engine crawling behavior.

### Social Media Functions

#### `applyOpenGraphBindings(config: OpenGraphConfig)`

Applies multiple Open Graph tags at once.

#### `applyTwitterCardBindings(config: TwitterCardConfig)`

Applies multiple Twitter Card tags at once.

#### `applyArticleBindings(config: ArticleConfig)`

Applies article-specific Open Graph tags.

#### `applySocialMediaBindings(config: SocialMediaConfig)`

All-in-one function for common social media tags.

### Configuration

#### `provideMetaConfig(config: MetaConfig)`

Configures global meta tag behavior.

## Best Practices

### ✅ DO

- Use `applySocialMediaBindings` for most use cases
- Keep descriptions between 150-160 characters
- Use high-quality images (1200x630px for Open Graph)
- Include `imageAlt` text for accessibility
- Set up robots directives per page/section

### ❌ DON'T

- Set the same meta tag multiple times unless it supports multi-instance
- Use low-resolution images for social previews
- Forget to set `og:type` (defaults to 'website')
- Include sensitive data in meta tags
- Manually manipulate meta tags when using bindings
