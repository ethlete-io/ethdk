# Color Theming — Design Specification

The color theming system lets components consume a named color palette without hardcoding any values. The palette is **project-specific** — consuming projects register their own themes. The SDK provides the infrastructure; the actual colors are defined per project.

---

## What is a color theme

A color theme is a named set of color swatches. Components pull from it via CSS custom properties. Switching the active theme switches all component colors simultaneously with no class or style changes on the components themselves.

### ThemeSwatch fields

<table>
  <tr>
    <td>Field</td>
    <td>Description</td>
  </tr>
  <tr>
    <td>`color`</td>
    <td>Primary color with per-state overrides: default, hover, focus, active, disabled</td>
  </tr>
  <tr>
    <td>`onColor`</td>
    <td>Text/icon color for use on a solid primary background, with per-state overrides</td>
  </tr>
  <tr>
    <td>`inkColor?`</td>
    <td>Optional accent color for ink, icons, and borders. Falls back to `color` when omitted.</td>
  </tr>
</table>

Color values are space-separated channel numbers — no `rgb()` wrapper:

- RGB: `"59 130 246"` (R G B, 0–255)
- HSL: `"217 91% 60%"` (H Hdeg% L%)

## Registration

Themes are registered once at application bootstrap:

```ts
provideColorThemesWithTailwind4([
  BRAND_THEME,
  DANGER_THEME,
  SUCCESS_THEME,
])
```

---

## Applying a theme

Pass the theme name or a `ColorTheme` object to any component that accepts a `color` input:

```html
<button et-button [etProvideColor]="'brand'">Label</button>
<button et-button [etProvideColor]="dangerTheme">Delete</button>
```

`[etProvideColor]` can also wrap a container — all themed children inside inherit its color context.

  </tr>

    <td>Raw RGB channels of ink</td>
  </tr>

### How opacity works

`--et-theme-color-primary-opacity` starts at `1`. Components override it per interaction state to produce tint layers. For example, a `tonal` button at rest sets it to `0.16`; at hover it becomes `0.24`. The base color never changes — only the opacity.

Interaction state management is automatic via `[etColorInteractive]`. Components do not manually track hover/focus/active — the directive drives the opacity.

## Figma mapping

Each color theme = one variable mode in Figma. Define variables for:

<table>
  <tr>
    <td>Figma variable</td>
    <td>Maps to</td>
  </tr>
  <tr>
    <td>`color/primary`</td>
    <td>`--et-theme-color-primary-solid`</td>
  </tr>
  <tr>
    <td>`color/on-primary`</td>
    <td>`--et-theme-color-on-primary-solid`</td>
  </tr>
  <tr>
    <td>`color/ink`</td>
    <td>`--et-theme-color-ink-solid`</td>
  </tr>
</table>

Tint layers in Figma are implemented as fills at the correct opacity — not as separate variables. The opacity values per variant and state are documented in each component's design spec.

Switching variable mode switches the entire component palette.
