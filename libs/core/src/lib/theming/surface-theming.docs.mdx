# Surface Theming — Design Specification

Surface themes define the background, text hierarchy, border, and interaction colors for a UI surface (page, card, panel, toolbar). Surfaces are elevation-aware — nested surfaces know their depth in the stack.

---

## What is a surface theme

A surface theme is a named set of tokens for a complete UI layer. Unlike color themes (which drive component accent colors), surface themes drive the canvas that components sit on: background, text, and borders. Components automatically adapt their ink colors to the active surface.

---

## SurfaceTheme structure

<table>
  <tr>
    <td>Field</td>
    <td>Type</td>
    <td>Description</td>
  </tr>
  <tr>
    <td>`name`</td>
    <td>`string`</td>
    <td>Identifier used to apply the surface (e.g. `'dark'`, `'light-elevated'`)</td>
  </tr>
  <tr>
    <td>`type`</td>
    <td>`'light' | 'dark'`</td>
    <td>Overall luminosity — used for automatic contrast decisions</td>
  </tr>
  <tr>
    <td>`elevation`</td>
    <td>`number`</td>
    <td>Stack depth: 0 = base, 1 = raised, 2 = higher. Components can request "next elevation" automatically.</td>
  </tr>
  <tr>
    <td>`isDefault?`</td>
    <td>`boolean`</td>
    <td>Applied when no explicit surface input is set</td>
  </tr>
  <tr>
    <td>`background`</td>
    <td>RGB</td>
    <td>Background fill color</td>
  </tr>
  <tr>
    <td>`color`</td>
    <td>RGB</td>
    <td>Primary text color</td>
  </tr>
  <tr>
    <td>`colorMuted`</td>
    <td>RGB</td>
    <td>Secondary / supporting text</td>
  </tr>
  <tr>
    <td>`colorSubtle`</td>
    <td>RGB</td>
    <td>Tertiary / placeholder text</td>
  </tr>
  <tr>
    <td>`border`</td>
    <td>RGB</td>
    <td>Divider and border color</td>
  </tr>
  <tr>
    <td>`interactionColor?`</td>
    <td>RGB per state</td>
    <td>Overlay color for interactive surfaces (hover, focus, active, disabled). Used by `[etSurfaceInteractive]`.</td>
  </tr>
</table>

Color values use space-separated channel numbers (same format as color themes): `"R G B"`.

---

## Registration

```ts
provideSurfaceThemesWithTailwind4([
  DARK_SURFACE,
  LIGHT_SURFACE,
  DARK_ELEVATED_SURFACE,
  DARK_ELEVATED_2_SURFACE,
])
```

---

## Applying a surface

```html
<div etProvideSurface="dark">...</div>
<div [etProvideSurface]="myCustomSurface">...</div>
```

The directive sets the surface context. All components inside inherit the surface tokens.

## Elevation nesting

Surfaces can be nested. The context tracker maintains the current elevation stack — a component inside a `dark` (elevation 0) surface can request elevation 1 and automatically receive `dark-elevated` without being explicitly told which surface to use.

In Figma: model elevation levels as separate frames/components with their own surface variable modes. Nesting a raised card inside a base page automatically applies the elevation-1 mode.
